Why it’s different

Most RAG demos are a black box you take on faith. This one is measured and inspectable.

📊 Measured

A real eval harness scores retrieval (hit-rate@k, MRR, precision@k) and answer faithfulness on a hand-built gold set. The improvement is proven, not asserted.

🔍 Inspectable

Every answer ships with the exact chunks it was built from, their source page, and relevance score. Click any citation to jump to its passage.

📈 Improved

Dense baseline → hybrid (BM25 + vector) → cross-encoder reranking, each step measured on the same gold set.

How it works

Built from scratch: chunking, embeddings, retrieval, grounded generation, citations.

Ingest: PDF / txt → parse → token-aware chunking → embed (Gemini) → store (ChromaDB) Ask: question → [ BM25 + vector retrieval → RRF fuse → cross-encoder rerank ] → top-k → grounded prompt (“answer ONLY from this context; cite the page; else say you don’t know”) → Gemini, streamed token-by-token → answer + clickable page citations + glass-box

Stack

  • Backend: Python · FastAPI (SSE streaming) · ChromaDB · Google Gemini
  • Retrieval: from-scratch chunking + cosine top-k · hybrid BM25 + vector (Reciprocal Rank Fusion) · cross-encoder reranking (sentence-transformers)
  • Frontend: Next.js / React · Tailwind CSS · streamed answers, clickable citations, glass-box, upload-with-progress · zero UI component libraries
  • Eval: custom harness with a gold Q&A set, page-level relevance, hit-rate@k / MRR / precision@k, plus faithfulness
